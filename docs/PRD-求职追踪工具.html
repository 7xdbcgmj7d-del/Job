二、产品概述

2.1 产品背景

产品实习生和应届毕业生在求职过程中面临以下痛点：

  • 投递混乱：投递岗位数量多，难以追踪每个岗位的投递状态
  • 信息分散：岗位信息、简历版本、面试安排分散在不同平台，管理困难
  • 复盘缺失：面试经验无法系统沉淀，重复踩坑
  • 进度不透明：缺乏直观的数据统计，无法把握求职全局

2.2 产品定位

面向产品实习生、应届毕业生的一站式求职管理工具，帮助用户高效追踪投递进度、管理简历版本、记录面试复盘，让求职过程有条不紊。

2.3 目标用户

用户画像特征描述
产品实习生寻找暑期/日常实习，投递量大，时间紧迫
应届毕业生秋招/春招求职，岗位类型多，需要精细化管理
转行求职者跨行业求职，需要管理不同方向的简历版本

2.4 核心价值

  • 效率提升：截图智能识别，快速录入岗位信息
  • 状态可视：看板视图 + 数据统计，投递进度一目了然
  • 知识沉淀：面试复盘记录，经验可复用
  • 隐私安全：数据本地存储，无需担心信息泄露

用户可以记录和管理所有投递的岗位信息，支持列表和看板两种视图模式。

3.2.2 功能点明细

功能点描述优先级
岗位列表以列表形式展示所有岗位，支持排序P0

3.2.3 岗位信息字段

公司文本是公司名称

工作类型枚举是全职/实习/兼职/合同工

5.2 核心操作流程

岗位添加流程

1. 点击"添加岗位"按钮
2. 弹出岗位表单对话框
3. 用户选择：
   a. 手动填写表单
   b. 上传截图 → AI识别 → 自动填充表单
4. 填写/修改完成后点击保存
5. 新岗位添加成功，跳转到岗位列表

截图识别流程

1. 用户拖拽或点击上传截图
2. 系统显示图片预览
3. 用户点击"智能识别"按钮
4. 系统调用AI API进行解析
5. 解析成功，自动填充表单字段
6. 用户可手动修改识别结果
7. 点击保存完成添加

七、风险与应对

风险影响应对措施
AI识别准确度不足用户需要大量手动修改提供手动填写备选，持续优化Prompt
浏览器存储空间不足无法存储大量数据提示用户定期清理，支持导出备份
API服务不稳定截图识别功能不可用支持多API切换，提供手动填写
用户忘记备份数据丢失风险定期提醒备份，开发自动备份功能
